# Bubble Sort

## Time Complexity

- Best: O(n)
- Average: O(n²)
- Worst: O(n²)

## Space Complexity

- O(1)

## Stable

Yes

## Notes

- Repeatedly compares adjacent elements.
- Swaps elements if they are in the wrong order.
- Largest elements gradually move toward the end.
- Simple and easy to implement.
- Stable sorting algorithm (preserves relative order of equal elements).
- In-place sorting algorithm (does not require extra memory).
- Performs poorly for large datasets.
- Useful for understanding sorting fundamentals.
- Can stop early using a swap flag optimization.
- Best case becomes O(n) when the array is already sorted.
- Performs more swaps compared to Selection Sort.
- Rarely used in production systems.
