# Insertion Sort

## Time Complexity

- Best: O(n)
- Worst: O(n²)

## Space Complexity

- O(1)

## Stable

Yes

## Notes

- Builds the sorted array one element at a time.
- Efficient for small or nearly sorted datasets.
- Performs well when the array is already partially sorted.
- In-place sorting algorithm (does not require extra memory).
- Stable sorting algorithm (preserves relative order of equal elements).
- More efficient than Bubble Sort in practical scenarios.
- Commonly used in hybrid sorting algorithms like TimSort.
- Worst-case occurs when the array is sorted in reverse order.
- Best-case occurs when the array is already sorted.
